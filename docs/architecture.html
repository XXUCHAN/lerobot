Robot Dataset Production Platform

Versioned robot data, sensor sync, reusable instruction annotations, reproducible training datasets

핵심 가치

로봇 데이터셋 생산에서 반복되는 원본 추적 불가, 센서 싱크 오류, instruction 증가에 따른 데이터 복제, 학습 재현성 부족을 플랫폼 계층으로 분리해 해결한다.

1. 원본 데이터 추적 불가

Cost: 재현 실패 조사 시간

원인: raw video/state/action이 폴더와 스크립트에 흩어짐
해결: Iceberg Snapshot으로 raw episode 고정
효과: dataset이 참조한 원본을 즉시 추적

2. 센서 프레임 싱크 오류

Cost: 재학습·디버깅 반복

원인: Camera / Joint / Action 주기가 서로 다름
해결: 기준 timestamp로 observation-action window 정렬
효과: 잘못된 학습 pair를 생성 단계에서 차단

3. action 데이터 표현 확장 비용 증가

Cost: 영상 복사·Dataset 재생성

원인: 표현 추가마다 영상 포함 dataset copy 생성
해결: episode는 1회 저장, instruction은 metadata 참조
효과: annotation/manifest만 증가

4. 학습 재현성 부족

Cost: 성능 회귀 원인 분석

원인: manifest, sync rule, annotation version이 흩어짐
해결: Registry가 dataset 생성 조건을 함께 고정
효과: 동일 dataset 재생성과 변경점 비교 가능
